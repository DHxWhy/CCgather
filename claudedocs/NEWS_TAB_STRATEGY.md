## 1. Executive Summary

### 1.1 CCgather News Tab 정의

| 항목 | 정의 |
|------|------|
| **역할** | AI 뉴스 매거진 (단순 애그리게이터 X) |
| **콘텐츠** | 팩트 기반 재창작 (원문 복사 X) |
| **가치** | CCgather만의 해석 + 실무 인사이트 |
| **경험** | CCgather에서 읽고, CCgather 링크 공유 |
| **목적** | 유입 → 가입 → 사용량 제출 유도 |

### 1.2 핵심 차별화 포인트

```
일반 뉴스 애그리게이터          CCgather News
─────────────────────────    ─────────────────────────
원문 링크로 이동               CCgather에서 읽기
단순 요약                      팩트 기반 재창작
누구나 타겟                    Claude Code 사용자 타겟
정보 전달만                    인사이트 + 실무 팁 제공
트래픽 유출                    트래픽 유지 + 전환
```

### 1.3 전체 퍼널 구조

```

## 2. 비즈니스 목표

### 2.1 최종 목표

```
뉴스 탭의 궁극적 목적:
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
  1. CCgather 회원 가입 유도
  2. Claude Code 사용량 제출 유도
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

### 2.2 목표 계층 구조

| 우선순위 | 목표 | 측정 지표 |
|----------|------|----------|
| **1순위** | 회원 가입 | 뉴스 → 가입 전환율 |
| **2순위** | 사용량 제출 | 가입 → 제출 전환율 |
| **3순위** | 재방문 유도 | 7일 내 재방문율 |
| **4순위** | 바이럴 확산 | 공유 클릭률, 유입 추적 |

### 2.3 뉴스 탭의 역할

```
뉴스 탭 = 유입 채널 (Acquisition Channel)

┌────────────────────────────────────────────────────────────────────┐
│                                                                    │
│  "뉴스로 가치를 제공하여 신뢰를 얻고,                               │
│   신뢰를 바탕으로 가입을 유도한다"                                   │
│                                                                    │
│  가치 제공 (뉴스) → 신뢰 형성 → 가입 유도 → 사용량 제출             │
│                                                                    │
└────────────────────────────────────────────────────────────────────┘
```

## 3. 콘텐츠 전략

### 3.1 본문 길이 정책

**원칙: 내용이 분량을 결정한다**

분량은 미리 정하지 않습니다. 팩트 전달과 맥락 설명에 필요한 만큼 자연스럽게 결정됩니다.

```
┌────────────────────────────────────────────────────────────────────┐
│  분량 결정 로직                                                     │
├────────────────────────────────────────────────────────────────────┤
│                                                                    │
│  Step 1: 팩트 추출                                                  │
│    → 필수 팩트 N개                                                  │
│    → 맥락 설명 필요 항목 M개                                        │
│                                                                    │
│  Step 2: 섹션별 필요 분량 산정                                       │
│    - 도입부: 2-3문장 (필수)                                         │
│    - 팩트당: 1-2문장                                                │
│    - 맥락 설명: 1-2문단/항목                                        │
│    - 인사이트: 1-2문단 (필수)                                       │
│    - 핵심 정리: 3-5포인트 (필수)                                    │
│                                                                    │
│  Step 3: 자연스러운 분량 도출                                        │
│    예시:                                                            │
│    - 팩트 5개 + 맥락 2개 → 약 800-1200자                            │
│    - 팩트 10개 + 맥락 5개 → 약 1500-2500자                          │
│                                                                    │
│  Step 4: 범위 제한 (품질 유지)                                       │
│    - 최소: 500자 (너무 짧으면 가치 부족)                            │
│    - 최대: 3000자 (너무 길면 집중도 저하)                           │
│                                                                    │
└────────────────────────────────────────────────────────────────────┘
```

### 3.2 필수 포함 요소

| 섹션 | 필수 여부 | 설명 |
|------|----------|------|
| **한 줄 요약** | ✅ 필수 | 공유용, 이모지 + 15자 내외 |
| **제목** | ✅ 필수 | CCgather 스타일, 명확하고 구체적 |
| **메타 정보** | ✅ 필수 | 날짜, 읽기 시간, 카테고리 |
| **도입부** | ✅ 필수 | 무슨 일인지, 왜 중요한지 (2-3문장) |
| **상세 내용** | ✅ 필수 | 팩트 기반 본문 (유동적 길이) |
| **CCgather 인사이트** | ✅ 필수 | 당신에게 의미하는 것 (1-2문단) |
| **핵심 정리** | ✅ 필수 | Key Takeaways (3-5포인트) |
| **관련 기사** | 선택 | 이전 관련 기사 링크 |
| **출처** | ✅ 필수 | 원문 출처 표기 |

## 4. 페르소나 및 톤앤매너

### 4.1 CCgather 뉴스 페르소나

```
┌────────────────────────────────────────────────────────────────────┐
│  CCgather = "친절한 동료 개발자"                                    │
├────────────────────────────────────────────────────────────────────┤
│                                                                    │
│  • Claude Code를 직접 사용하는 개발자                               │
│  • 최신 정보를 빠르게 파악하고 동료에게 공유하는 사람                │
│  • 기술적이지만 어렵지 않게 설명하는 커뮤니케이터                    │
│  • "우리"의 언어를 사용하는 커뮤니티 멤버                           │
│                                                                    │
└────────────────────────────────────────────────────────────────────┘
```

### 4.2 톤앤매너 가이드

| 항목 | 권장 | 지양 |
|------|------|------|
| **인칭** | "당신", "우리" | "사용자", "고객" |
| **어조** | 친근하고 전문적 | 딱딱하거나 가벼움 |
| **설명** | 맥락과 함께 | 용어 나열만 |
| **가치** | "이게 왜 중요한지" | 사실 나열만 |
| **표현** | 구체적, 실용적 | 추상적, 과장 |

### 4.3 톤앤매너 예시

```
❌ 지양하는 표현:
"Anthropic이 Claude Code 1.0.38을 발표했습니다.
터미널 성능이 개선되었습니다."

✅ 권장하는 표현:
"Claude Code가 1.0.38로 업데이트됐어요.
이번 버전의 핵심은 터미널 성능 개선인데요,
대용량 파일 작업할 때 체감 속도가 확 달라질 거예요."
```

### 4.4 CCgather 인사이트 작성 가이드

```
인사이트 = "당신에게 이게 의미하는 것"

포함해야 할 내용:
1. 실무에서 어떻게 활용할 수 있는지
2. 주의해야 할 점이 있는지
3. 추천하는 액션이 있는지

예시:
┌────────────────────────────────────────────────────────────────────┐
│  💡 CCgather 인사이트                                               │
│  ───────────────────────────────────────────────────────────────   │
│  이번 업데이트는 대용량 프로젝트를 다루는 분들에게 특히              │

│  확인할 때 답답함이 확 줄어들 거예요.                                │
│                                                                    │
│  💡 Tip: 업데이트 후에는 터미널 세션을 새로 시작해야                 │
│  성능 개선이 적용돼요. 기존 세션에서는 효과가 없을 수 있어요.        │
└────────────────────────────────────────────────────────────────────┘
```

## 5. 콘텐츠 선정 기준

### 5.1 카테고리별 선정 기준

| 카테고리 | 선정 기준 | 키워드 |
|----------|----------|--------|
| **Claude Code News** | 직접 관련, 실무 영향 | Claude Code, Claude CLI, Anthropic API, Claude 모델 |
| **AI News** | 개발자 생산성, 실무 활용 | AI 코딩 도구, LLM 실무, 개발자 워크플로우 |

### 5.2 우선순위 체계

| Priority | 유형 | 예시 | 발행 속도 |
|----------|------|------|----------|
| 🔴 **P0** | Claude Code 직접 관련 | 버전 업데이트, 새 기능, 버그 수정 | 즉시 |
| 🟠 **P1** | Anthropic 생태계 | Claude API, 모델 업데이트, 가격 변경 | 당일 |
| 🟡 **P2** | AI 코딩 도구 전반 | Cursor, Copilot, 경쟁사 동향 | 1-2일 |
| 🟢 **P3** | 개발자 생산성 | LLM 활용 팁, 워크플로우 개선 | 적절한 때 |

### 5.3 제외 대상

| 제외 유형 | 이유 |
|----------|------|
| 일반 AI 연구 논문 | 타겟과 관련 없음 |
| AI 윤리/철학 토론 | 실무 영향 없음 |
| 투자/비즈니스 뉴스 | 개발자 관점 아님 |
| 루머/미확인 정보 | 신뢰도 저하 |
| 홍보성 콘텐츠 | 가치 없음 |

### 5.4 필터 질문

```
콘텐츠 선정 시 반드시 물어볼 질문:
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  "이 뉴스를 읽은 Claude Code 사용자가
   내일 업무에서 뭔가 다르게 할 수 있는가?"

  → Yes: 발행
  → No: 제외

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

## 6. 팩트 검증 시스템

### 6.1 검증 원칙

```

### 6.4 팩트 검증 DB 스키마

```sql
-- 기사별 팩트 저장
CREATE TABLE article_facts (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  content_id UUID REFERENCES contents(id) ON DELETE CASCADE,
  fact_type VARCHAR(50) NOT NULL,    -- version, metric, date, feature, entity, action, quote
  original_text TEXT NOT NULL,       -- 원문에서 추출한 텍스트
  fact_value TEXT NOT NULL,          -- 정규화된 팩트 값
  is_required BOOLEAN DEFAULT true,  -- 필수 팩트 여부
  is_verified BOOLEAN DEFAULT false, -- 검증 완료 여부
  verified_in_text TEXT,             -- 재창작 본문에서 매칭된 텍스트
  created_at TIMESTAMPTZ DEFAULT NOW()
);

-- contents 테이블 확장
ALTER TABLE contents ADD COLUMN fact_score INTEGER DEFAULT 0;
ALTER TABLE contents ADD COLUMN fact_verified_at TIMESTAMPTZ;
```

function ShareSection({ article }: { article: Article }) {
  const handleCopy = async (text: string) => {
    await navigator.clipboard.writeText(text);
    toast.success('복사되었습니다');
  };

  const handleShare = (platform: 'x' | 'linkedin') => {
    const urls = {
      x: `https://x.com/intent/tweet?text=${encodeURIComponent(xShareText(article))}`,
      linkedin: `https://www.linkedin.com/sharing/share-offsite/?url=${encodeURIComponent(article.url)}`,

    window.open(urls[platform], '_blank');

ALTER TABLE contents ADD COLUMN fact_verified_at TIMESTAMPTZ;
ALTER TABLE contents ADD COLUMN read_time VARCHAR(10);
ALTER TABLE contents ADD COLUMN category VARCHAR(50);

-- 팩트 테이블
CREATE TABLE article_facts (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  content_id UUID REFERENCES contents(id) ON DELETE CASCADE,
  fact_type VARCHAR(50) NOT NULL,
  original_text TEXT NOT NULL,
  fact_value TEXT NOT NULL,
  is_required BOOLEAN DEFAULT true,
  is_verified BOOLEAN DEFAULT false,
  verified_in_text TEXT,
  created_at TIMESTAMPTZ DEFAULT NOW()
);

CREATE INDEX idx_article_facts_content_id ON article_facts(content_id);

-- 관련 기사 테이블
CREATE TABLE related_articles (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  content_id UUID REFERENCES contents(id) ON DELETE CASCADE,
  related_content_id UUID REFERENCES contents(id) ON DELETE CASCADE,
  relevance_score DECIMAL(3,2),
  created_at TIMESTAMPTZ DEFAULT NOW(),
  UNIQUE(content_id, related_content_id)
);

  prompt: string;
  score: number;  // AI 유사도 점수
}

function ImageSelector({ articleId, candidates }: Props) {
  const [selected, setSelected] = useState<string | null>(null);

  const handleConfirm = async () => {
    await replaceArticleImage(articleId, selected);
    toast.success('이미지가 교체되었습니다');
  };

  return (
    <div className="grid grid-cols-3 gap-4">
      {candidates.map((img) => (
        <div
          key={img.id}
          className={cn(
            "cursor-pointer border-2 rounded-lg",
            selected === img.id ? "border-orange-500" : "border-white/10"
          )}
          onClick={() => setSelected(img.id)}
        >
          <img src={img.url} alt="candidate" />
          <p className="text-xs text-white/50">유사도: {img.score}%</p>
        </div>
      ))}
      <Button onClick={handleConfirm} disabled={!selected}>
        이 이미지로 교체
      </Button>
    </div>
  );
}
